# Research - 问题调研与信息收集

## 命令用途

对技术问题、方案选择、最佳实践等进行深入调研，当仅靠代码仓库无法解答问题时，通过多种途径（MCP、网络搜索、文档查询等）收集信息并输出结论。

**核心原则：**

- **多渠道调研**：优先使用代码仓库，必要时调用 MCP 或网络搜索
- **信息验证**：交叉验证多个来源的信息，确保准确性
- **结论导向**：直接输出调研结论，不保存中间文档
- **不修改代码**：仅进行调研，不执行代码修改

## 使用方式

```
/research <调研问题或主题>
```

示例：

```
/research React 18 中 useEffect 的清理函数最佳实践
/research Hono 框架在 Cloudflare Workers 中的错误处理模式
/research Zustand 状态管理库的最新版本特性和迁移指南
```

## 执行流程

### 第一步：理解调研需求

- 明确调研问题的核心目标
- 识别需要解答的具体问题
- 确定调研的范围和深度

### 第二步：代码仓库内调研

优先在代码仓库中查找相关信息：

- 搜索相关代码文件和实现
- 查看项目文档和 README
- 检查配置文件和使用示例
- 分析现有实现模式和约定

### 第三步：外部资源调研

当代码仓库无法提供足够信息时，使用外部资源：

**MCP 资源（如适用）：**

- 使用 Context7 MCP 查询库文档和代码示例
- 查询官方文档和最佳实践
- 获取最新的 API 参考和迁移指南

**网络搜索：**

- 搜索技术文档和官方指南
- 查找社区讨论和最佳实践
- 参考开源项目实现
- 查阅技术博客和教程

**文档查询：**

- 查看项目内的规则文件（`.cursor/rules/`）
- 参考架构文档和设计文档
- 查阅相关 PRD 和测试用例

### 第四步：信息整合与分析

- 汇总来自不同来源的信息
- 交叉验证信息的准确性和时效性
- 识别关键结论和最佳实践
- 区分官方推荐方案和社区实践

### 第五步：输出调研结论

直接输出调研结果，包括：

- **核心结论**：明确回答调研问题
- **关键信息**：重要的技术细节和注意事项
- **最佳实践**：推荐的实现方式和模式
- **参考来源**：关键信息来源（如适用）
- **注意事项**：潜在风险和限制

**输出要求：**

- 结论清晰、可操作
- 避免冗长的中间过程描述
- 重点突出关键信息
- 不保存为文档文件

---

## 调研策略

### 技术问题调研

- 优先查看项目现有实现
- 查询官方文档和 API 参考
- 搜索社区最佳实践
- 对比多个方案的优缺点

### 方案选择调研

- 列出可行的技术方案
- 对比各方案的优缺点
- 评估与项目现有架构的适配度
- 考虑实现成本和维护成本

### 最佳实践调研

- 查询官方推荐做法
- 参考知名项目的实现
- 了解社区共识
- 注意版本差异和迁移路径

---

## 注意事项

1. **不修改代码**：此命令仅用于调研，不执行任何代码修改
2. **不保存文档**：调研结论直接输出，不保存为文档文件
3. **信息准确性**：优先使用官方文档和权威来源
4. **时效性**：注意信息的时效性，优先使用最新版本的信息
5. **项目上下文**：结合项目实际情况，提供针对性的建议
